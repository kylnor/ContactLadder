**WHAT (Text + Visual)**

**What Is a Buyer Profile?**

A Buyer Profile is a saved set of criteria that defines what a buyer is looking for. Cloze uses this to automatically match MLS listings.

**Matching Criteria You Can Set:**

| Category | Fields | Example |
|----------|--------|---------|
| **Price** | Min/Max price range | $400k - $550k |
| **Property Details** | Beds, baths, sqft, lot size | 3+ beds, 2+ baths, 1800+ sqft |
| **Location** | Cities, neighborhoods, zip codes | Summerlin, Henderson, 89135 |
| **Features** | Pool, garage, stories, HOA | Pool, 2-car garage, single-story |
| **Property Type** | Single family, condo, townhome | Single family homes only |
| **Custom Fields** | Any MLS field you want to track | Schools, view, gated, etc. |

**How Matching Works:**

Once you create a buyer profile:
1. Cloze searches your MLS automatically
2. New matching listings appear in the buyer's profile
3. You get notified when new matches arrive
4. You curate and send the best options to the buyer
5. Track opens and clicks to see engagement

**The Curated Property Alert:**

Instead of forwarding raw MLS links, Cloze lets you:
- Select multiple matching properties
- Add a personal note to each
- Send a beautifully formatted email with photos
- Track opens and link clicks
- See which properties they're most interested in

- Q: "Can I save a search without linking it to a specific buyer?"

**WHAT (Text + Visual)**

**What Is a Transaction in Cloze?**

A Transaction (also called a Property, Project, or Deal) is a record that tracks:
- The property being bought/sold
- The buyer(s) and seller(s) involved
- Other parties (agents, lender, title, inspector)
- Transaction timeline (offer, contingencies, close date)
- Documents, notes, and communications
- Deal Team (who's collaborating on this transaction)

**Transaction Stages:**

| Stage | What It Means | What You Track |
|-------|--------------|----------------|
| **Pending** | Offer submitted, not yet accepted | Offer amount, terms, response deadline |
| **In Contract** | Accepted offer, moving toward close | Contingencies, inspections, appraisal, loan approval |
| **Clear to Close** | All contingencies removed, awaiting closing | Final walkthrough, closing date |
| **Closed** | Deal done, keys handed over | Close date, final sale price, commission |
| **Lost** | Deal fell apart or client chose another agent | Reason, follow-up plan |

**Next Steps for Transactions:**

You can customize Next Steps for each transaction type:

**Buyer Transaction Checklist Example:**
1. Submit offer
2. Negotiate and get acceptance
3. Earnest money deposited
4. Order home inspection (due in 10 days)
5. Inspection contingency decision (due in 14 days)
6. Appraisal ordered (due in 21 days)
7. Loan approval (due in 25 days)
8. Final walkthrough (day before close)
9. Closing day!
10. Post-close follow-up (3 days after close)

**Seller Transaction Checklist Example:**
1. Listing agreement signed
2. Photos and staging completed
3. MLS entry live
4. First open house
5. Offer received
6. Negotiate and accept offer
7. Inspection (buyer's 10-day window)
8. Repairs negotiation
9. Appraisal (buyer's lender)
10. Clear to close
11. Closing day!

**WHAT (Text + Visual)**

**How the Matching Engine Works:**

The matching engine compares buyer profiles to MLS listings using a weighted scoring system:

```
BUYER PROFILE CRITERIA:
- Required fields (must match exactly)
- Preferred fields (scored, but not required)
- Weighted fields (some criteria matter more than others)
   â†“
MATCHING ENGINE ALGORITHM:
- Searches MLS for listings meeting required criteria
- Scores properties based on preferred fields
- Ranks matches by total score
- Presents top-scoring properties first
   â†“
OUTPUT:
- High-quality matches (90-100% score)
- Good matches (70-89% score)
- Partial matches (50-69% score)
```

**Matching Field Types:**

| Field Type | How It Works | Example | When to Use |
|-----------|-------------|---------|-------------|
| **Required** | Property MUST match to appear | "MUST have pool" | Non-negotiable criteria |
| **Preferred** | Property scores higher if it matches | "Prefers mountain views" | Nice-to-haves |
| **Range** | Property must fall within range | "$400k-$550k" | Flexible on exact number |
| **Weighted** | Some fields count more than others | "Location = 2x weight" | Prioritize what matters most |
| **Exclude** | Property must NOT have this | "No HOA" | Deal-breakers |

**Matching Score Breakdown:**

Cloze assigns each match a score:

- **90-100%:** Perfect match (meets all required + most preferred criteria)
- **70-89%:** Good match (meets all required + some preferred criteria)
- **50-69%:** Partial match (meets required, missing most preferred)
- **<50%:** Weak match (meets minimum requirements only)

**Common Matching Challenges:**

1. **Too Many Matches:** Criteria too broad, need tighter filters
2. **Too Few Matches:** Criteria too narrow, need to relax requirements
3. **Wrong Matches:** Required vs Preferred misconfigured
4. **Missing Matches:** MLS data fields incomplete, need alternate search strategy

**HOW to Optimize Matching (Video - 5-6 min)**

**WHAT IF (Text)**

- Q: "What if the matching engine shows properties that are clearly wrong (wrong city, wrong price)?"
  - A: This is usually an MLS data error, not a Cloze error. The listing agent may have entered the wrong city or price in the MLS. Report to your MLS admin.

- Q: "What if I want to exclude certain subdivisions or streets?"
  - A: Most MLS integrations allow exclusion filters. Add "Exclude subdivision: [name]" in the profile. If not available, manually filter those out when reviewing matches.

- Q: "What if my buyer's criteria change mid-search?"
  - A: Update the profile immediately. Matching results refresh automatically. Old matches that no longer fit will drop off, new matches will appear.

- Q: "Can I see WHY a property matched?"
  - A: Some integrations show match score breakdown (price: 100%, beds: 100%, pool: 0%, etc.). If yours doesn't, you can infer by comparing listing details to buyer criteria.

- Q: "What if a buyer says 'just send me everything in Henderson under $500k'?"
  - A: Create the profile with those broad criteria, but set up a weekly digest instead of daily alerts. You can manually curate the best 3-5 from the weekly batch instead of overwhelming them with 50 properties.

- Q: "What if I want to match based on school district, not just city?"
  - A: If your MLS has school district as a searchable field, add it to the profile. If not, you'll need to manually verify school districts on matches before sending.

- Q: "Can I match expired listings or off-market properties?"
  - A: The matching engine only works with active MLS listings. For off-market, you'd need to manually add those properties to Cloze and tag them for specific buyers.

- Q: "What if my buyer wants a property that doesn't exist in the current market?"
  - A: Keep the profile active. When a matching property hits the MLS, you'll be notified immediately. This is perfect for buyers with very specific criteria waiting for the right home.
